# Contributing to LSCamoflash

Thank you for your interest in contributing to LSCamoflash.

This project focuses on **local-first, non-invasive firmware extensions** for IP cameras.
Please read the guidelines below to ensure efficient collaboration.

---

## Scope of Contributions

We welcome contributions in the following areas:
- Bug fixes
- Security improvements
- Build system enhancements
- Documentation improvements
- Support for additional compatible devices
- Performance and stability improvements

Out of scope:
- Changes to original vendor firmware
- Cloud integrations or dependencies
- Malicious functionality or surveillance misuse

---

## How to Contribute

### 1. Issues
- Use GitHub Issues for:
  - Bug reports
  - Feature requests
  - Compatibility reports
- Please provide:
  - Device model and hardware revision
  - LSCamoflash version
  - Steps to reproduce (if applicable)
  - Logs or error messages (if available)

**Do not report security vulnerabilities via public issues.**  
See `SECURITY.md` for responsible disclosure.

---

### 2. Pull Requests

Before submitting a PR:
- Fork the repository
- Create a feature branch
- Keep changes focused and minimal
- Ensure the build completes successfully

PR requirements:
- Clear description of the change
- Reference related issues (if any)
- No breaking changes without prior discussion

---

## Coding Guidelines

- Prefer readability over cleverness
- Avoid unnecessary dependencies
- Keep changes compatible with embedded/low-resource environments
- Follow existing project structure and conventions

---

## Licensing

---

## Communication

- Be respectful and constructive
- Keep discussions technical and on-topic
- Decisions are made based on security, stability, and project goals

---

Thank you for helping improve LSCamoflash.
