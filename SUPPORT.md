# Support

This document describes how to get help with LSCamoflash
and what kind of support is (and is not) provided.

---

## Supported Scope

We provide support for:
- Installation issues related to LSCamoflash
- Bugs in the LSCamoflash codebase
- Build system problems
- Documentation issues
- Compatibility reports for supported devices

Support is provided **on a best-effort basis**.

---

## Unsupported Scope

We do **not** provide support for:
- Original vendor firmware
- Cloud services or vendor backends (e.g. TUYA cloud)
- Unauthorized device modification
- Reverse engineering requests beyond this project
- Custom forks or heavily modified installations
- Legal advice or interpretation of laws
- Use cases that violate the Acceptable Use Policy

Requests related to unsupported scope may be closed without response.

---

## How to Get Support

### 1. GitHub Issues (Preferred)

For bugs and feature requests, please open a GitHub Issue:
https://github.com/Bunkerschild/LSCamoflash/issues

When opening an issue, include:
- Device model and hardware revision
- LSCamoflash version
- Clear description of the problem
- Steps to reproduce (if applicable)
- Relevant logs or error messages

---

### 2. Security Issues

**Do not report security vulnerabilities via GitHub Issues.**

Please follow the responsible disclosure process described in:
- `SECURITY.md`

---

### 3. Questions and Discussions

General questions or technical discussions should use:
- GitHub Discussions (if enabled)

Please avoid using Issues for general questions.

---

## Response Time

This is an open-source project maintained in limited free time.

- There is **no guaranteed response time**
- Issues may be prioritized based on:
  - Security relevance
  - Reproducibility
  - Impact on stability
